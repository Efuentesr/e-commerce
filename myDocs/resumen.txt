Aplicación lista para ejecutar                                                                                        
  Levantar todo con Docker:                                                                                             
  docker compose up --build

  Eso:
  1. Levanta PostgreSQL con health check
  2. Ejecuta migraciones automáticamente
  3. Carga 112 productos en 10 categorías con imágenes
  4. Inicia backend en http://localhost:8000
  5. Inicia frontend en http://localhost:3000

  Qué se implementó

  Backend (Django):
  - 3 apps: accounts, products, sales
  - Máquina de estados de órdenes con control de stock (separated_qty)
  - Auditoría de precios con django-simple-history
  - 112 productos con 3 imágenes Pillow cada uno
  - JWT con refresh automático

  Frontend (React + Tailwind):
  - UI tipo Amazon (navbar, grid multi-columna, sidebar de categorías)
  - Carrito persistente (localStorage para invitados → sincroniza al login)
  - Drawer de carrito con generación de pedido
  - Página de órdenes con botón Enviar por WhatsApp
  - Panel de admin: editar descuentos, marcar pagada, anular
  - Registro/login de usuarios



  Sí, puedo ayudarte a desplegarlo. Las opciones más prácticas para este stack:

  ---
  Opción 1 — Railway (la más simple para este proyecto)
  - Soporta Docker Compose directamente
  - PostgreSQL incluido como servicio
  - Free tier disponible
  - Despliegue desde GitHub o CLI

  Opción 2 — Render
  - Backend Django como Web Service
  - Frontend React como Static Site
  - PostgreSQL administrado
  - Free tier con limitaciones (se "duerme" tras inactividad)

  Opción 3 — Fly.io
  - Muy cercano a Docker, buen control
  - Free tier generoso
  - Requiere algo más de configuración

  ---
  Mi recomendación: Railway

  Es la que menos fricción tiene para un proyecto con docker-compose.yml ya configurado. El mayor trabajo sería:

  1. Separar las variables de entorno para producción (.env.production)
  2. Configurar el VITE_API_URL del frontend para apuntar al backend desplegado
  3. Ajustar ALLOWED_HOSTS y CORS_ALLOWED_ORIGINS en Django
  4. Manejar los archivos de medios (imágenes) — probablemente con un bucket S3 o Cloudflare R2

  Cuando retomes mañana arrancamos directamente con eso. Hasta entonces!
